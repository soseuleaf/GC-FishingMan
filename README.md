# GC_FishingMan
22-1 컴퓨터 그래픽스 과제

## 개요
섬에서 낚시를 하는 남자를 주제로 가진 프로그램이다. WASD 버튼을 통해 섬을 이동할 수 있으며, 스페이스를 누르면 찌를 던져 낚시를 시작한다. 찌가 흔들릴 때 스페이스를 한 번 더 누르면 낚시대를 들어 올리며 낚은 물체를 자랑한다. 찌가 흔들리지 않을 때 스페이스를 눌렀다면, 아무것도 낚이지 않는다.

## 구현 기능
1. 장면에 등장하는 물체는 glutSolid... 를 포함한다. glutWire... 도 일부 사용해도 좋다.
    - 낚는 물체는 sphere, cube, torus, cone, cube, teapot 중에 랜덤으로 나오며, 물고기 형태의 물체도 낚을 수 있다. 물고기는 sphere과 cone을 적절하게 사용하여 만들었다.

2. 물체는 timer callback 을 이용하여 시간에 따라 스스로 움직인다.
    - motionInfoHandler는 timer callback에 따라 큐에 들어있는 MotionInfo 구조체의 데이터를 해석하여 해당 값을 tick에 대비하여 시간에 따라 회전되는 값이나 움직이는 값을 계산하여 linkedval에 해당 값을 집어넣는다. 해당 MotionInfo의 tick이 마지막에 도달하면 연결된 MotionInfo으로 넘어가 계산을 계속하고, tick이 마지막에 도달하지 않았다면 tick을 1 증가시키고 자신을 다시 큐에 집어넣는 방식으로 값들이 시간에 따라 변화되게 만들었다. 
대표적으로 해와 달은 glRotate를 통해 0부터 360도까지 값이 증가하는데, MotionInfo sunmoon_1를 참조하여 변경되는 float sunmoonRotate 값을 통해 돌아간다.

3. 장면에 두 개 이상의 다관절 물체가 등장한다. 두 물체의 구조는 서로 다르다.
    - 사람, 낚싯대, 물고기가 다관절 물체로 등장한다.

4. 적어도 하나의 다관절 물체는 2개 이상의 분기가 있어야 한다. 즉, (tree 구조로 해석한다면) 어떤 부모 물체에는 2개 이상의 자식 물체가 연결되어 있어야 한다.
    - 사람은 몸통-왼팔, 오른팔, 오른다리, 왼다리 총 4개의 분기가 연결되어 있다.

5. 적어도 하나의 다관절 물체는, 중심 물체로부터 3단계 이상의 연결이 있어야 한다. 즉, (tree 구조로 해석한다면) depth 가 3이상이어야 한다.
    - 낚싯대는 낚싯대-낚싯줄-찌-(사용자 조작에 따라 물고기가 연결됨.) 4단계의 연결이 되어있다.

6. 조명 기능이 포함되어야 한다.
    - 0, 0 좌표에서 360도로 돌아가는 해와 달은 조명 기능을 포함하고 있다.

## 보너스 조건
1. 다관절 물체의 움직임이, 무엇인가 현실에 있는 물체/상황을 흉내내어 움직임이 그럴 듯하다.
    - 사용자의 스페이스바에 따라 찌를 던지거나 회수하는 작업과, 신나서 만세하는 모션이 구현되어 있다.

2. 일부 물체는 키보드/마우스 조작에 따라 연동하여 움직인다.
   - 사람은 WASD를 통해 이동할 수 있으며, 스페이스바를 통해 추가적인 모션을 수행한다.

3. 서로 다른 2개 이상의 재질 (색상)이 등장한다.
   - 여러 색을 넣었다.

4. 사용자 반응성이 있다.
예1: 주인공 캐릭터가 동전 근처에 가면(먹으면) 동전이 사라짐.
예2: 적 캐릭터는 주인공 캐릭터를 자동으로 추격함/도망감 (절대 거리가 가까워지는/멀어지는 방향으로 움직임)
    - 사용자가 fishing 상태일 때 0.01초마다 1/1000 확률로 계산하여 hit 상태를 활성화시키며 찌를 흔들리게 만든다. 여기서 1초 이내로 사용자가 스페이스를 한 번 더 누르면 물고기가 낚아진다. 1초 이내로 스페이스를 누르지 않았거나, 찌가 흔들리지 않았다면, 아무것도 낚이지 않는다.
